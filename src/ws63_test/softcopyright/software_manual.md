# 《WS63双板激光打标控制软件》软件说明书

## 1. 文档概述

### 1.1 编写目的

本文档用于说明《WS63双板激光打标控制软件》的功能范围、系统架构、运行环境、核心流程、主要模块、数据设计和部署方式，可作为中国软件著作权登记时的软件说明材料，也可用于项目归档和技术交接。

### 1.2 软件名称

- 软件全称：`WS63双板激光打标控制软件`
- 软件简称：`WS63激光打标控制软件`
- 版本号：`V1.0`

### 1.3 适用场景

本软件适用于需要上位机 G-Code 指令输入、无线控制指令转发、振镜轨迹执行、激光功率控制和安全停光保护的激光打标设备。

## 2. 开发背景

传统激光打标控制方案中，上位机指令接入、控制逻辑处理和执行端驱动常常集中在单一控制端，或依赖较多有线连接。在实际设备交付中，这类方案容易受到布线复杂、调试不便和异常链路安全保护不足等问题的影响。

本软件采用基于 WS63 的双板式控制架构。发射板负责接入上位机指令并进行 G-Code 解析，接收板负责命令执行、振镜控制和激光输出，两板之间通过 SLE 星闪无线链路协同工作。该设计提高了部署灵活性，也使控制链路更容易拆分调试、维护和扩展。

## 3. 开发目标

本软件的开发目标主要包括：

1. 支持通过上位机或串口工具输入常用 G-Code 控制指令。
2. 将文本指令转换为可无线传输的结构化运动控制命令。
3. 实现发射板和接收板之间带校验、带 ACK 的可靠命令传输。
4. 在接收板侧完成运动插补，并输出振镜控制坐标。
5. 根据加工指令控制激光启停和功率输出。
6. 提供实时状态回传和链路超时安全保护能力。

## 4. 运行环境

### 4.1 硬件环境

- 一套 WS63 发射板
- 一套 WS63 接收板
- 与上位机连接的 UART 通讯接口
- 用于振镜坐标输出的 DAC8562 芯片
- 用于激光功率控制的 PWM 输出通道
- 激光与振镜执行机构

### 4.2 软件环境

- WS63 固件编译环境
- 工程所使用的 LiteOS / OSAL 运行环境
- LaserGRBL、串口助手或其它上位机指令发送软件

### 4.3 当前接口基线

根据项目当前配置，软件使用以下接口基线：

- 上位机业务串口：`UART1`
- 默认串口波特率：`115200`
- DAC 接口：`SPI0`
- 激光控制输出：`PWM2`
- 双板间无线链路：`SLE`

## 5. 总体架构

本软件由共享协议层、发射板控制层和接收板执行层三部分构成。

整体数据流如下：

```text
上位机 / LaserGRBL
    -> UART 文本 G-Code
    -> 发射板
    -> G-Code 解析与命令组包
    -> SLE 无线下发
    -> 接收板
    -> 命令队列与插补执行
    -> DAC X/Y 输出 + PWM 激光控制
    -> 状态回传与 ACK 反馈
```

其中，发射板主要负责理解上位机指令并维持上位机兼容交互；接收板主要负责执行控制、硬件输出和安全保护。

## 6. 功能设计

### 6.1 共享协议与配置模块

共享模块定义了发射板和接收板之间的统一数据契约，主要职责包括：

- 定义运动命令包结构
- 定义状态反馈包结构
- 定义命令类型和状态码
- 定义 CRC16 校验逻辑
- 定义队列深度、超时参数、引脚映射和流控参数

该模块保证了双板之间在命令格式、状态格式和时序控制上的一致性。

### 6.2 发射板命令处理模块

发射板软件通过 UART 接收上位机发送的文本 G-Code，并识别运动、坐标模式切换、原点设置、进给速度调整和激光功率调整等控制指令，随后将其转换为结构化运动命令包。

主要职责包括：

- 初始化业务串口
- 按行接收上位机指令
- 支持本地状态查询命令
- 维护命令序号
- 输出与上位机兼容的 `ok` 和状态响应

### 6.3 无线通信模块

无线通信模块由发射板侧 SLE 客户端和接收板侧 SLE 服务端组成。

发射板侧负责：

- 扫描接收板目标设备
- 建立无线连接
- 进行服务发现和特征值发现
- 下发命令包
- 接收状态包和队列余量信息

接收板侧负责：

- 接收无线写请求
- 校验包长和 CRC
- 校验命令合法性
- 记录序号 ACK 信息
- 返回状态和队列余量

### 6.4 接收板执行模块

接收板是本软件的实际执行端。它将校验通过的命令写入队列，再由插补和硬件控制模块顺序消费执行。

主要职责包括：

- 对有效命令进行排队缓存
- 按顺序读取控制命令
- 将运动命令转换为插补输出
- 维护当前 X/Y 位置
- 协调执行状态和队列状态回传

### 6.5 插补与运动控制模块

对 `G0`、`G1` 等运动命令，本软件采用线性插补方式，使当前点按设定速度逐步运动到目标点。插补过程中会持续计算中间位置，并将坐标转换为 DAC 数值，从而驱动振镜执行轨迹。

该模块具备以下特点：

- 支持绝对坐标和相对坐标模式
- 区分快速移动和进给移动
- 维护当前位置和目标位置状态
- 长路径执行过程中周期性让出调度机会，避免任务长时间占用 CPU

### 6.6 激光控制模块

激光控制模块根据控制命令管理激光输出状态。

主要功能包括：

- 初始化 PWM 输出资源
- 控制激光启停
- 将激光功率值转换为 PWM 占空比
- 在停光或触发安全保护时强制将功率清零

### 6.7 安全监控模块

安全监控模块用于防止通信异常或任务活动中断导致激光继续失控输出。

主要功能包括：

- 跟踪最近一次 SLE 通信时间
- 跟踪最近一次命令活动时间
- 区分空闲态和活动态超时门限
- 当连续超时达到确认条件后，强制停光并清空队列

该模块是设备安全链路中的重要组成部分。

## 7. 任务与线程设计

### 7.1 发射板侧任务

发射板主要包含以下任务：

1. UART 接收任务：负责接收上位机指令并协调处理。
2. SLE 初始化任务：负责完成无线扫描、连接和服务发现。
3. 心跳任务：定时发送心跳包，维持接收板安全看门狗。

### 7.2 接收板侧任务

接收板主要包含以下任务：

1. 插补任务：消费命令队列并执行运动控制。
2. 安全任务：检查超时条件并触发保护。
3. SLE 初始化任务：初始化接收板无线服务。

## 8. 主要业务流程

### 8.1 系统启动流程

1. 发射板初始化 G-Code 处理器和 UART 接口。
2. 发射板启动无线初始化任务和心跳任务。
3. 接收板初始化 DAC、PWM、插补模块、安全监控模块和命令队列。
4. 接收板启动插补任务、安全任务和无线服务初始化任务。
5. 无线连接建立成功后，系统进入可控状态。

### 8.2 命令执行流程

1. 上位机通过 UART 发送一行 G-Code 指令。
2. 发射板解析该指令，并生成一条或多条运动命令包。
3. 发射板通过 SLE 将命令包写入接收板。
4. 接收板完成包校验并将有效命令放入队列。
5. 接收板返回最新 ACK 序号和队列剩余容量。
6. 插补任务从队列中读取命令，并驱动振镜运动或激光输出。
7. 接收板周期性返回当前位置和运行状态。
8. 发射板利用回传状态响应上位机查询，并据此进行流控。

### 8.3 安全保护流程

1. 安全任务周期性检查无线通信是否超时。
2. 判断当前系统处于空闲态还是活动态。
3. 当连续超时达到确认阈值后，立即关闭激光输出。
4. 清空命令队列，防止异常恢复后继续执行旧命令。

## 9. 数据设计

### 9.1 运动命令包

软件定义了统一的运动命令结构，典型字段包括：

- 命令类型
- 控制标志位
- 序列号
- 目标 X 坐标
- 目标 Y 坐标
- 进给速度
- 激光功率
- CRC16 校验值

该数据结构使发射板和接收板能够以固定二进制格式交换运动、模式切换和激光控制命令。

### 9.2 状态反馈包

状态反馈包用于接收板返回以下信息：

- 当前运行状态
- 错误码
- 最新确认的 ACK 序号
- 队列剩余容量
- 当前 X/Y 位置

该数据结构是上位机状态查询、节流控制和联调观察的重要基础。

### 10.2 `transmitter`

### 10.3 `receiver`

## 11. 对外接口说明

### 11.1 上位机命令接口

本软件通过 UART 接收文本形式的 G-Code 指令，支持常见运动控制和激光控制命令，并返回命令确认和状态信息。

### 11.2 无线控制接口

双板之间通过自定义 SLE 服务和特征值交换运动命令和执行状态。

### 11.3 硬件输出接口

接收板使用以下硬件接口：

- 通过 SPI 与 DAC 芯片通信
- 通过 DAC 输出振镜 X/Y 控制坐标
- 通过 PWM 输出激光功率控制信号

## 12. 可靠性与安全性设计

本软件通过多种机制共同提高运行可靠性：

- CRC16 包校验
- 队列余量反馈
- ACK 序号跟踪
- 在途写请求限制与重试策略
- 独立心跳保活机制
- 空闲态与活动态双超时门限
- 异常条件下强制停光

这些措施有助于降低实际设备运行过程中的命令丢失、队列过载和链路异常风险。

## 13. 技术特点与创新点

相较于基础的单链路运动控制实现，本软件具有以下特点：

1. 形成了从 G-Code 文本输入到振镜与激光执行的完整控制链路。
2. 通过双板架构实现上位机协议处理与执行端控制解耦。
3. 将无线命令传输、ACK 回传和队列流控结合在一起。
4. 在接收端执行路径中集成插补、激光控制和超时安全保护。
5. 工程结构清晰，便于后续联调、测试和二次开发。
