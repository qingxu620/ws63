## 2. 软件概述

### 2.1 软件用途

本软件用于激光打标设备控制场景，能够接收来自上位机或串口工具发送的 G-Code 文本指令，经发射板解析并转换为结构化控制命令，再通过 SLE 星闪无线链路发送至接收板。接收板在收到命令后，对数据进行校验、排队、插补计算和执行控制，最终输出振镜坐标和激光功率控制信号，实现打标路径运动和激光出光控制。

### 2.2 软件特点

本软件具备以下特点：

1. 采用双板式控制架构，将上位机协议处理与执行控制解耦。
2. 支持 UART 文本指令接入，兼容常见 G-Code 控制方式。
3. 使用 SLE 星闪无线链路完成双板命令通信与状态反馈。
4. 支持运动插补、激光开关、激光功率调节和状态查询。
5. 具备命令序号、CRC 校验、队列控制和超时安全保护机制。
6. 适合用于功能开发、联调验证、现场演示和项目交付。

### 2.3 软件组成

本软件主要由以下三部分组成：

- 共享模块 `common`：负责协议结构、全局参数、CRC 校验和硬件基线配置。
- 发射板模块 `transmitter`：负责接收上位机 G-Code、做指令解析、协议转换和无线发送。
- 接收板模块 `receiver`：负责命令校验、命令排队、插补执行、激光控制和安全监控。

### 2.4 典型应用链路

典型应用链路如下：

```text
上位机 / LaserGRBL / 串口工具
        |
        | UART 文本指令
        v
   WS63 发射板
        |
        | SLE 无线控制命令
        v
   WS63 接收板
        |
        +--> DAC8562 输出振镜 X/Y
        +--> PWM 输出激光功率
        |
        `--> 状态回传到发射板
```

## 3. 运行环境与准备条件

### 3.1 硬件环境

在安装和使用本软件前，应准备以下硬件资源：

- 一块刷写发射固件的 WS63 开发板
- 一块刷写接收固件的 WS63 开发板
- 一台上位机电脑
- 一根连接发射板业务串口的串口线
- 可选的两路调试串口线，用于查看发射板和接收板的启动日志
- 一套激光振镜执行机构
- 一路 DAC8562 输出电路
- 一路基于 PWM 的激光功率控制电路

### 3.2 软件环境

建议准备以下软件环境：

- 项目编译环境
- Python 运行环境
- 串口调试工具或 LaserGRBL
- 固件烧录工具
- 文档整理工具，例如 Word 或 WPS

### 3.3 当前工程接口基线

根据工程默认配置，当前接口基线如下：

#### 发射板接口

- 业务串口：UART1
- UART TX：GPIO15
- UART RX：GPIO16
- 串口波特率：115200

#### 接收板接口

- DAC 通信总线：SPI0
- DAC 片选引脚：GPIO10
- DAC 时钟引脚：GPIO7
- DAC 数据输出引脚：GPIO9
- 激光 PWM 输出：GPIO2
- PWM 通道：PWM2

### 3.4 使用前检查项

开始操作前，建议逐项确认以下内容：

1. 两块板子的固件角色没有刷反。
2. 发射板和接收板均已成功上电。
3. 上位机已连接到发射板业务串口，而不是调试串口。
4. 激光器和振镜连接线正确，且硬件供电稳定。
5. 若现场需要安全演示，应先断开激光输出或将功率设为最小。
6. 现场操作人员已明确紧急断电方式和激光安全防护要求。

## 4. 软件安装与部署

### 4.1 构建目标选择

本软件在一个工程中同时维护发射板和接收板两个固件目标。部署前，需要分别构建两种固件。

进入工程根目录后，可通过如下方式进入配置界面：

```bash
python3 build.py menuconfig ws63-liteos-app
```

在配置界面中，应按实际需要二选一：

- 构建接收板固件时，开启 `CONFIG_LASER_MARKER_RECEIVER`
- 构建发射板固件时，开启 `CONFIG_LASER_MARKER_TRANSMITTER`

注意事项如下：

1. 不建议同时开启这两个选项。
2. 每次切换固件角色后，建议重新执行一次完整编译。
3. 如出现历史构建缓存干扰，可先执行清理构建。

### 4.2 编译方法

配置完成后，在工程根目录执行：

```bash
python3 build.py ws63-liteos-app
```

如需清理后重新编译，可执行：

```bash
python3 build.py -c ws63-liteos-app
python3 build.py ws63-liteos-app
```

### 4.3 输出文件说明

编译完成后，常用输出文件位置如下：

- 固件包：`output/ws63/fwpkg/ws63-liteos-app/ws63-liteos-app_all.fwpkg`
- 调试符号文件：`output/ws63/acore/ws63-liteos-app/ws63-liteos-app.elf`

在现场部署中，通常使用固件包进行烧录；在问题排查和调试分析中，调试符号文件用于定位代码问题。

### 4.4 烧录顺序建议

建议按以下顺序部署：

1. 先将接收板固件烧录到接收板。
2. 再将发射板固件烧录到发射板。
3. 烧录完成后，分别上电并观察串口启动日志。
4. 确认两板均初始化正常后，再连接业务串口和上位机。

### 4.5 首次部署建议

首次部署时，建议采用以下原则：

- 先只连接调试串口，确认启动日志正常。
- 再连接业务串口，确认发射板能够响应上位机命令。
- 最后接入激光执行机构，进行低功率联调。

## 5. 软件启动与初始化操作

### 5.1 接收板启动过程

接收板主要负责硬件初始化、命令接收、运动插补和激光控制。上电后，会依次完成：

1. DAC 初始化
2. 激光 PWM 控制初始化
3. 插补模块初始化
4. 安全监控模块初始化
5. 命令队列初始化
6. 插补任务创建
7. 安全任务创建
8. SLE 服务初始化任务创建

如果启动正常，日志中通常可看到类似信息：

- `WS63 Laser Marker - Receiver Board`
- `[receiver] hardware init OK`
- `[receiver] all tasks created`
- `[laser rx] SLE enable called`

### 5.2 发射板启动过程

发射板主要负责上位机命令接入、G-Code 解析、无线发送和心跳维护。上电后，会依次完成：

1. G-Code 处理器初始化
2. 业务串口初始化
3. UART 接收任务创建
4. SLE 初始化任务创建
5. 心跳任务创建

如果启动正常，日志中通常可看到类似信息：

- `WS63 Laser Marker - Transmitter Board`
- `[transmitter] init OK`
- `[transmitter] all tasks created`
- `[laser tx] SLE enable called`

### 5.3 启动完成后的状态判断

当两块板子都启动成功后，应进一步确认以下状态：

1. 发射板是否成功扫描到接收板。
2. 发射板和接收板是否建立无线连接。
3. 发射板是否已经发现接收板命令和状态特征值。
4. 上位机发送 `?` 指令时是否能收到状态字符串。

若上述条件均满足，说明系统已进入可操作状态。

## 6. 基本操作方法

### 6.1 连接上位机

将上位机业务串口连接到发射板 UART1 对应的接口。连接完成后，打开串口工具或 LaserGRBL，并按以下参数设置：

- 波特率：115200
- 数据位：8
- 停止位：1
- 校验位：无

连接成功后，发射板通常会返回兼容 Grbl 风格的欢迎信息或在收到命令时做出响应。

### 6.2 最小操作流程

建议首次操作按以下流程执行：

1. 上电发射板和接收板。
2. 确认两板启动日志无明显报错。
3. 确认发射板与接收板已建立 SLE 连接。
4. 通过串口工具向发射板发送简单查询命令。
5. 再发送低风险运动指令。
6. 最后再测试激光开关和功率。

### 6.3 常用命令类别

本软件支持以下常见命令类别：

- 信息查询命令，如 `$I`
- 状态查询命令，如 `?`
- 坐标模式切换命令，如 `G90`、`G91`
- 原点设置命令，如 `G92`
- 快速移动和线性移动命令，如 `G0`、`G1`
- 激光控制命令，如 `M3`、`M4`、`M5`
- 速度和功率参数命令，如 `F`、`S`

### 6.4 首次联调推荐命令

在串口工具中，可按行发送以下命令进行最小联调：

```gcode
$I
G90
M3 S200
G1 X10 Y10 F6000
G1 X20 Y10
M5
?
```

### 6.5 预期现象

执行上述命令时，预期现象如下：

1. 每条合法命令返回 `ok`。
2. 接收板按照目标轨迹驱动振镜运动。
3. `M3` 命令开启激光，`M5` 命令关闭激光。
4. `?` 命令返回当前状态信息，例如 `<Idle|...>` 或 `<Run|...>`。

### 6.6 日常操作顺序建议

在正常使用中，建议遵循以下顺序：

1. 上电并等待双板连接稳定。
2. 发送基础初始化命令，例如坐标模式和功率参数命令。
3. 发送空走命令验证坐标控制。
4. 确认运动无误后，再开启激光。
5. 加工结束后，先关闭激光，再断开业务连接和设备电源。

## 7. 详细功能操作说明

### 7.1 查询软件信息

当操作人员希望确认当前系统是否正常工作时，可发送信息查询指令。常用方法如下：

- 发送 `$I`：用于查看软件标识信息
- 发送 `?`：用于查看运行状态

建议在以下场景使用：

- 设备刚上电时
- 更换固件版本后
- 现场出现异常、需要快速确认运行状态时

### 7.2 坐标模式切换

本软件支持绝对坐标模式和相对坐标模式。

- `G90`：表示绝对坐标模式
- `G91`：表示相对坐标模式

操作建议如下：

1. 如现场操作人员对当前坐标状态不确定，优先切回 `G90`。
2. 批量调试局部移动时，可根据需要使用 `G91`。
3. 在正式加工前，建议统一设置一次坐标模式，避免轨迹偏移。

### 7.3 原点设置

通过 `G92` 命令可设置当前坐标为指定原点。该功能适用于：

- 工件重新定位
- 调试过程中临时归零
- 加工前对参考点进行校准

操作原点设置时，应先确认振镜当前位置与实际工艺位置一致，再进行原点设定。

### 7.4 运动控制

本软件支持以下运动方式：

- `G0`：快速移动，通常用于不出光的定位
- `G1`：线性插补移动，通常用于加工路径执行

常见操作方法如下：

```gcode
G90
G0 X0 Y0
G1 X10 Y10 F6000
G1 X20 Y5 F5000
```

说明如下：

1. `X`、`Y` 表示目标坐标。
2. `F` 表示进给速度。
3. 快速移动通常不用于正式出光。
4. 加工移动应在轨迹和功率确认后再执行。

### 7.5 激光控制

激光控制通常通过以下命令完成：

- `M3` 或 `M4`：开启激光
- `M5`：关闭激光
- `S`：设置激光功率

示例如下：

```gcode
M3 S100
G1 X5 Y5 F3000
M5
```

操作建议如下：

1. 首次测试时，建议从较低功率值开始。
2. 在仅验证轨迹时，可先不发送 `M3`。
3. 停止加工前，应先执行 `M5` 关闭激光。

### 7.6 状态查看

发射板收到接收板回传状态后，可通过 `?` 指令返回状态文本。状态信息通常可用于观察：

- 当前是否空闲
- 当前是否正在运行
- 当前反馈坐标
- 命令确认情况
- 队列剩余情况

在现场调试中，建议养成“发送关键动作前后都查一次状态”的习惯，以便及时发现异常。

## 9. 日志查看与状态判断

### 9.1 为什么要看日志

本软件运行在双板系统中，很多问题不是单看上位机回包就能定位。为了提高排障效率，建议同时观察发射板和接收板日志。

### 9.2 发射板常见日志含义

发射板日志一般用于判断以下内容：

- 串口初始化是否成功
- SLE 扫描和连接是否成功
- 命令是否成功发送
- 心跳是否正常工作
- 状态句柄是否准备完成

常见日志示例如下：

- `[transmitter] init OK`
- `[laser tx] SLE enable called`
- `[laser tx] scanning...`
- `[laser tx] heartbeat send fail ...`

### 9.3 接收板常见日志含义

接收板日志一般用于判断以下内容：

- 硬件初始化是否成功
- SLE 服务是否成功启动
- 命令是否通过长度和 CRC 校验
- 命令队列是否已满
- 安全监控是否触发停光保护

常见日志示例如下：

- `[receiver] hardware init OK`
- `[laser rx] service started`
- `[laser rx] CRC error`
- `[laser rx] queue full`
- `Laser OFF`

### 9.4 状态异常时的基本判断逻辑

建议按如下逻辑判断：

1. 若两板都没正常启动，优先看固件角色和硬件连接。
2. 若两板正常启动但不连接，优先看 SLE 扫描和设备名。
3. 若发射板能收命令但接收板不动，优先看 CRC、队列和命令校验。
4. 若振镜运动正常但激光无输出，优先看 `M3/M5`、`S` 和 PWM 配置。
5. 若运行一段时间后停光，优先看安全超时和无线稳定性。

## 10. 常见问题与处理办法

### 10.1 上位机无响应

可能原因：

- 串口接错到了 UART0 调试口
- 波特率设置错误
- 发射板未正常启动

处理办法：

1. 确认连接的是发射板 UART1。
2. 确认波特率为 115200。
3. 重新上电并查看发射板日志。

### 10.2 发射板和接收板无法建立连接

可能原因：

- 接收板未成功开启 SLE 服务
- 发射板未扫描到接收板
- 固件角色刷写错误

处理办法：

1. 检查两块板是否烧录正确固件。
2. 检查接收板日志中是否有服务启动信息。
3. 检查发射板日志中是否有扫描和连接信息。

### 10.3 指令返回 `ok` 但设备不动作

可能原因：

- 接收板未真正收到有效命令
- 命令校验失败
- 命令队列已满

处理办法：

1. 查看接收板是否有长度错误或 CRC 错误日志。
2. 查看是否出现 `queue full`。
3. 重新用简单命令做最小验证。

### 10.4 振镜动作正常但激光不出光

可能原因：

- 未发送 `M3` 或发送后功率为 0
- 激光 PWM 引脚配置与实际硬件不一致
- 安全监控触发了停光

处理办法：

1. 检查是否发送了 `M3 Sxxx`。
2. 检查 PWM 输出硬件连接。
3. 查看接收板是否有超时停光日志。

### 10.5 运行途中突然停光

可能原因：

- 无线心跳中断
- 通信抖动导致安全保护触发
- 长时间业务阻塞

处理办法：

1. 查看接收板是否打印超时保护相关日志。
2. 检查发射板心跳发送是否正常。
3. 检查现场无线环境和供电稳定性。

## 11. 安全注意事项

### 11.1 激光安全

本软件用于激光打标设备控制，操作时必须遵守激光安全规范。建议做到以下几点：

1. 操作人员佩戴符合要求的激光防护设备。
2. 在调试阶段优先使用低功率或空走模式。
3. 在未确认轨迹前，不要直接执行高功率打标。
4. 发生异常时，应优先关闭激光输出并切断设备电源。

### 11.2 通信安全

双板依赖无线链路协同工作，若出现异常，应避免继续长时间加工。建议做到以下几点：

1. 开机后先确认无线连接稳定。
2. 连续加工前先做一次最小轨迹验证。
3. 若频繁出现超时停光，应暂停使用并排查原因。

### 11.3 操作安全

建议遵循以下操作规范：

1. 每次加工前先检查原点和坐标模式。
2. 每次加工后先关闭激光，再断电。
3. 不得在不清楚当前状态的情况下连续发送大量高功率命令。
4. 不得在设备有明显异常日志时继续强行运行。
