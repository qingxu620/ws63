# WS63 控制张大头 Emm_V5.0 工程说明

这份说明对应新增的 `ws63_test/zdt_controller/` 工程。它的定位不是替代现有 `receiver/` 或 `transmitter/`，而是给后续 `感知与对焦节点` 里的 `Z` 轴执行器先落一个独立、可编译、可扩展的 WS63 基线。

如果你已经明确本赛季 `Z` 轴走 `TTL 串口`，并希望把它并入正式的“感知与对焦节点”，现在优先使用 `ws63_test/focus_node/` 角色；`zdt_controller/` 更适合单独 bring-up 和底层协议调试。

## 1. 当前工程覆盖的能力

已经实现的最小控制集：

- 电机使能 / 失能 `0xF3`
- 速度模式 `0xF6`
- 位置模式 `0xFD`
- 立即停止 `0xFE`
- 同步开始 `0xFF`
- 设置单圈回零零点 `0x93`
- 清零当前位置 `0x0A`
- 解除堵转保护 `0x0E`
- 触发回零 `0x9A`
- 强制退出回零 `0x9C`
- 读取固件/硬件版本 `0x1F`
- 读取电机状态 `0x3A`
- 读取回零状态 `0x3B`
- 读取实时转速 `0x35`
- 读取实时位置 `0x36`
- 读取系统参数总入口 `Read_Sys_Params`
- 修改开环/闭环控制模式 `0x46`
- 修改回零参数 `0x4C`
- 官方 `Emm_V5.h/.c` 风格兼容封装

## 2. 当前工程没有一次性做满的部分

为了保证第一版是稳的，这次仍然没有把所有驱动参数都做成“结构化解析”，尤其是：

- `0x42 / 0x43` 回复载荷的完整结构体拆解
- 上电自动运行参数 `0xF7`
- 回零参数读取类命令的全量封装

也就是说，当前版本已经能“发对命令、收对回包、兼容官方 STM32 API 风格”，但对于部分读回参数，仍以原始帧缓存或上层二次解析为主。

## 3. 串口/RS485 接线建议

默认 `menuconfig` 里给的是：

- UART: `UART1`
- TX: `GPIO15`
- RX: `GPIO16`
- Baud: `115200`

如果使用 TTL 串口模块，直接接 `TX/RX/GND` 即可。当前感知与对焦节点第一版也默认采用这一路径。  
如果使用 RS485 模块，除了 `A/B/GND` 外，建议启用 `CONFIG_ZDT_RS485_DIR_ENABLE`，再额外接一根 `DE/RE` 方向控制脚。

## 4. 代码结构

```text
zdt_controller/
├── Emm_V5.c         # 官方 STM32 API 风格兼容层
├── Emm_V5.h         # 官方 STM32 API 风格头文件
├── main.c            # 板级入口与演示任务
├── zdt_uart.c        # UART / RS485 半双工收发
├── zdt_protocol.c    # 张大头自定义协议打包/解析
└── zdt_controller.c  # 高层同步控制接口
```

## 5. 协议依据

当前 WS63 版本协议实现依据：

- 仓库内 `ZDT/Emm_V5.0步进闭环驱动说明书Rev1.3.pdf`
- 仓库内 `ZDT/STM32_串口通讯__位置模式/BSP/Emm_V5.c`
- 仓库内 `ZDT/STM32_串口通讯__位置模式/BSP/Emm_V5.h`
- 张大头公开说明文章：
  `https://blog.csdn.net/zhangdatou666/article/details/132644047`

当前移植策略不是直接照搬 STM32 外设驱动，而是：

- 以官方 `Emm_V5.c/.h` 的命令集和接口语义作为对照基准
- 用 WS63 的 `UART + pinctrl + osal` 重写底层收发
- 在 `Emm_V5.c/.h` 中复刻官方风格调用入口
- 在 `zdt_protocol/zdt_controller` 中保留更适合 WS63 工程化复用的底层实现
