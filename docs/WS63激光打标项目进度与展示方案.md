# WS63 激光打标项目进度与展示方案

## 1. 当前总体进度

本项目已经完成三条激光打标控制链路的阶段性落地，形成了从最小闭环、网络链路到星闪结构化任务链路的完整演进路径。

| 方向 | 工程目录 | 上位机/客户端 | 当前状态 | 价值 |
| --- | --- | --- | --- | --- |
| 串口有线直连 RX | `src/ws63_laser_single/` | LaserGRBL | 已成功落地 | 验证 G-code 解析、运动执行、DAC 输出、激光 PWM 的最小稳定闭环 |
| WiFi SoftAP TCP 直连 RX | `src/ws63_laser_wifi/` | LaserGRBL | 已成功落地 | 验证 WS63 作为 SoftAP + TCP Grbl endpoint 的无线控制能力 |
| 星闪 SLE 结构化任务链路 | `src/ws63_laser_sle_job/` + `src/ws63_laser_sle_job_host/` | 自研 Host 上位机 | 已成功打通 | 验证 TX/RX 分板、SLE 任务包、RX 本地缓存执行、预缓冲边收边执行 |

这三条链路不是互相替代的孤立 demo，而是一个逐步降低不确定性的技术路线：

```text
串口单板闭环
  -> 验证激光打标核心控制能力

WiFi TCP 单板闭环
  -> 验证网络无线接入和 LaserGRBL 兼容路径

星闪 TX/RX 任务链路
  -> 验证多节点、低功耗短距无线、结构化任务传输和本地安全执行
```

## 2. 已完成能力梳理

### 2.1 `ws63_laser_single`：串口有线直连 RX

已完成：

- LaserGRBL 通过 UART 直接连接 WS63 RX。
- RX 侧兼容常用 Grbl 握手和基础 G-code。
- 已打通 G-code parser、motion executor、DAC8562 X/Y 输出、激光 PWM。
- 单板独立完成解析和执行，是所有后续链路的控制基准。

该方向适合作为现场展示的“最稳基准链路”：如果需要证明激光运动和出光本身是可靠的，优先展示这一链路。

### 2.2 `ws63_laser_wifi`：WiFi SoftAP TCP 直连 RX

已完成：

- WS63 RX 作为 SoftAP。
- PC 连接 WS63 热点后，LaserGRBL 通过 TCP/Telnet 连接。
- TCP 字节流进入 Grbl 解析路径，执行本地运动和激光控制。
- 断开连接时具备强制安全停止方向。

该方向适合作为“通用无线接入能力”展示：观众可以直观看到无需 USB 串口线，LaserGRBL 仍能控制 WS63 激光设备。

### 2.3 `ws63_laser_sle_job`：星闪结构化任务链路

已完成：

- PC 自研 Host 通过 USB UART 连接 TX。
- TX 将任务封装为 SLE 结构化协议包。
- RX 接收 `JOB_BEGIN / JOB_DATA / JOB_END / EXEC_START` 等任务包。
- RX 进行 seq、offset、CRC、job_id、状态校验。
- RX 将 G-code 写入本地 job cache，再由本地 executor 执行。
- 预缓冲模式已成功打通：RX 可以在任务尚未完全接收时提前启动执行。
- Host 已能监听 RX 日志并识别 `[JOB_EXEC] done`，更新 UI 状态为执行完成。

星闪方向当前已经从“能传输”推进到“能稳定展示”的阶段，是后续作品成品化的主线。

## 3. 三种链路的定位

| 链路 | 定位 | 展示重点 | 后续是否继续保留 |
| --- | --- | --- | --- |
| 串口直连 | 技术基线 | 运动和激光控制稳定性 | 保留，作为回归测试和安全基准 |
| WiFi TCP | 通用网络方案 | LaserGRBL 无线控制 | 保留，作为对比方案和备用无线链路 |
| 星闪 SLE | 作品主线 | 多节点协同、任务传输、安全执行、自研 UI | 重点完善，作为最终成品方向 |

最终展示时建议不要把三条链路讲成“重复做了三版”，而是讲成：

1. 先用串口单板验证控制核心。
2. 再用 WiFi 验证网络接入能力。
3. 最后用星闪做成具有产品形态的多节点系统。

## 4. 星闪成品系统设想

最终成品建议组织为三个星闪节点：

```text
PC / 自研上位机
      |
      | USB UART
      v
TX 发射节点
      |
      | SLE 结构化任务包
      v
RX 激光执行节点  <---- SLE 状态/控制 ---->  中控屏节点
      |
      v
DAC X/Y + Laser PWM + 运动执行
```

### 4.1 RX 激光执行节点

职责：

- 作为真正的执行端和安全端。
- 接收 SLE job 包。
- 校验任务完整性和顺序。
- 本地缓存 G-code。
- 执行 motion 和 laser control。
- 任何异常、断连、取消、停止都以 RX 为准强制关光。

展示关键词：

- “执行权在 RX”
- “激光安全由 RX 兜底”
- “任务接收和运动执行解耦”

### 4.2 TX 发射节点

职责：

- 连接 PC 上位机。
- 接收 Host 下发的任务。
- 分包、编号、等待 ACK、处理 NACK。
- 将任务通过 SLE 可靠送达 RX。

展示关键词：

- “PC 到星闪网络的网关”
- “结构化任务包，不是透明串口桥”
- “可诊断、可重传、可确认”

### 4.3 中控屏节点

职责设想：

- 展示设备状态：空闲、接收中、执行中、完成、错误、暂停、急停。
- 展示任务信息：文件名、大小、进度、行数、预计/实际耗时。
- 提供本地控制：开始、暂停、停止、复位、回零、低功率预览。
- 展示连接状态：TX/RX/Host/SLE 信号状态。
- 提供更像产品的现场交互入口。

中控屏节点可以先从状态显示做起，不急于直接参与核心控制。第一阶段建议只做“监听/显示/发控制命令”，不要让它和 Host 同时成为两个独立 Grbl 控制器。

## 5. 最终上位机形态

当前 `src/ws63_laser_sle_job_host/` 已经从调试工具逐渐具备产品上位机雏形。最终建议打造成一个“完善且好看的星闪激光打标上位机”。

### 5.1 核心页面

建议主界面分为 5 个区域：

1. 顶部连接栏
   - TX 命令串口选择
   - TX 日志串口选择
   - RX 日志串口选择
   - 连接状态
   - SLE 状态

2. 左侧任务区
   - 打开 G-code 文件
   - 文件名、大小、CRC
   - G-code 文本预览
   - 普通模式 / 预缓冲模式选择
   - preroll 参数

3. 中间作业控制区
   - 上传任务
   - 上传并执行
   - 执行
   - 停止
   - 放弃任务
   - 查询状态

4. 右侧状态卡片
   - 当前状态：空闲 / 接收中 / 执行中 / 完成 / 错误
   - 上传进度
   - 执行行数
   - RX cache 空闲空间
   - 最终坐标
   - 最近错误

5. 底部日志区
   - 展示模式：只显示关键事件
   - 诊断模式：显示完整 TX/RX/Host 日志
   - 一键保存日志

### 5.2 UI 展示风格

建议风格：

- 工业控制台风格，不做花哨营销页。
- 信息密度适中，重点突出状态和动作按钮。
- 默认展示模式日志简洁：

```text
STATUS 上传配置: mode=preroll ...
TX日志 [JOB_TX_BEGIN] ...
RX日志 [JOB_BEGIN] ...
STATUS PREROLL_TRIGGER ...
RX日志 [JOB_EXEC] start ...
STATUS JOB_READY ...
RX日志 [JOB_EXEC] done ...
STATUS EXEC_DONE job=1 lines=1270 ...
```

- 诊断模式保留完整原始日志，便于复现问题。
- 错误、NACK、TIMEOUT、SAFE_STOP、ABORT 永远突出显示。

### 5.3 上位机最终能力清单

建议最终版本具备：

- 文件打开、编辑、预览。
- 自动 CRC。
- 普通上传执行。
- 预缓冲边收边执行。
- 三路日志监听：TX 命令、TX debug、RX debug。
- 自动识别执行完成。
- 保存完整日志包。
- 设备连接状态可视化。
- 任务状态可视化。
- 一键生成测试报告。
- 后续可扩展简单路径预览。

## 7. 作品展示组织方式

### 7.1 展台硬件摆放

建议分成四个区域：

```text
[PC 上位机]
   |
   | USB
   v
[TX 星闪发射节点]  --->  SLE  --->  [RX 激光执行节点]  --->  [激光打标机构]
                                      ^
                                      |
                                  [中控屏节点]
```

每块板建议贴清晰标签：

- TX：PC 网关 / 星闪发射
- RX：激光执行 / 安全控制
- SCREEN：本地状态显示 / 控制面板
- LASER：DAC + PWM + 振镜/运动机构

### 7.2 演示顺序

建议现场演示按下面节奏：

1. 展示串口直连版
   - 说明这是核心控制基线。
   - LaserGRBL 直接控制 WS63 RX。

2. 展示 WiFi 版
   - PC 连接 WS63 SoftAP。
   - LaserGRBL 通过 TCP 控制。
   - 说明 WS63 已具备无线接入能力。

3. 展示星闪版
   - 打开自研 Host。
   - 连接 TX/RX 日志。
   - 上传 G-code。
   - 选择预缓冲模式。
   - RX 提前开始执行。
   - Host 显示 `EXEC_DONE`。
   - 中控屏显示状态同步。

4. 讲安全设计
   - RX 是执行真相源。
   - 断连、停止、取消、错误都强制关光。
   - 不依赖 Host UI 状态判断激光是否关闭。

### 7.3 展示话术重点

可以这样概括：

> 我们不是只做了一个能动的激光 demo，而是把激光控制核心、WiFi 直连链路、星闪多节点任务链路都跑通了。最终成品会是 PC 上位机 + TX 星闪网关 + RX 激光执行器 + 中控屏的完整系统。RX 负责安全执行，TX 负责可靠任务传输，中控屏负责现场状态和控制，PC 上位机负责文件管理、任务下发和诊断。

## 8. 当前推荐主线

后续建议以星闪结构化任务链路作为主线，串口和 WiFi 作为支撑与对照：

- `ws63_laser_single`：保留为控制核心回归测试。
- `ws63_laser_wifi`：保留为无线直连对照方案。
- `ws63_laser_sle_job`：继续做稳定性、状态同步和安全闭环。
- `ws63_laser_sle_job_host`：继续做 UI 产品化。
- `ws63_screen_st7796_ft6336`：中控屏节点 bring-up 目录，目录名沿用历史 ST7796 版本；当前屏幕选型已切换到 `MSP3223/` 资料包中的 3.2 寸 ILI9341V + FT6336U 模块，后续需要把屏幕驱动从历史 ST7796 代码迁移到 MSP3223/ILI9341V。

最终目标是一个可现场展示、可讲清楚技术路线、也具备产品雏形的 WS63 星闪激光打标系统。
